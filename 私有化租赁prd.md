# 私有化租赁产品需求文档(PRD)

## 1. 产品概述

### 1.1 产品定位
私有化租赁是一款面向企业客户的AI能力集成服务，提供稳定、安全的本地部署解决方案，支持多种AI能力和业务场景的快速集成与应用。

### 1.2 核心价值
- **安全可控**：本地部署，数据不出企业边界
- **灵活扩展**：支持并发路数、场景、渠道的灵活扩展
- **快速上线**：标准化部署流程，降低实施成本
- **长期价值**：3年后系统归属客户，持续提供技术支持

## 2. 功能需求

### 2.1 AI能力模块

| 能力名称 | 功能描述 |
|---------|---------|
| TTS（文本转语音） | 将文本内容转换为自然流畅的语音输出 |
| ASR（语音识别） | 将语音内容转换为文本，支持多种语言和口音 |
| 身份证识别 | 自动识别身份证正反面信息，支持图像预处理和信息提取 |
| 人脸在框 | 检测人脸是否在指定区域内，用于身份验证场景 |
| 人脸对比 | 对比两张人脸图像的相似度，用于身份验证和识别 |

### 2.2 渠道支持

| 渠道类型 | 功能描述 |
|---------|---------|
| H5自助 | 基于移动端浏览器的自助服务渠道 |
| H5&PC(远程) | 同时支持移动端和PC端的远程服务渠道 |
| H5&移动端(远程) | 支持移动端应用和H5的远程服务渠道 |
| PC端(临柜) | 面向柜台工作人员的PC端服务渠道 |

### 2.3 业务场景

| 场景类型 | 应用说明 |
|---------|---------|
| 信贷 | 客户身份验证、信贷信息采集与审核 |
| 理财 | 客户风险评估、理财产品推荐与签约 |
| 保险 | 保险理赔资料审核、客户身份验证 |
| 其他 | 支持根据客户需求定制开发的业务场景 |

## 3. 技术要求

### 3.1 部署架构
- **私有化部署**：支持本地服务器部署，不依赖公网
- **容器化方案**：采用Docker容器化部署，简化安装与维护
- **可扩展性**：支持集群部署，满足高并发需求

### 3.2 系统要求
- **硬件要求**：8核CPU、16GB内存、500GB存储（基础配置）
- **操作系统**：支持CentOS 7.0+、Ubuntu 16.04+
- **网络要求**：内部网络访问，支持HTTP/HTTPS协议

### 3.3 性能指标
- **并发支持**：基础配置支持5路并发，可扩展至100路以上
- **响应时间**：AI能力调用响应时间<500ms
- **可用性**：系统可用性>99.9%

## 4. 服务内容

### 4.1 实施服务
- **系统部署**：完成私有化环境的安装与配置
- **功能定制**：根据客户需求进行必要的功能定制开发
- **测试验证**：进行全面的功能测试和性能测试
- **人员培训**：提供系统操作和维护培训

### 4.2 技术支持
- **电话支持**：工作日8:30-17:30电话技术支持
- **远程支持**：提供远程协助服务
- **问题响应**：一级问题2小时响应，4小时解决；二级问题4小时响应，8小时解决

## 5. 定价方案

### 5.1 基础套餐
- **合同期限**：3年起
- **基础费用**：7.5万/年
- **包含内容**：
  - 5路并发支持
  - 1个业务场景
  - 1个服务渠道
  - 全部AI能力模块
  - 年度技术支持与版本更新

### 5.2 扩展费用
- **并发扩展**：1.5万/路/年
- **场景扩展**：2万/场景/年
- **渠道扩展**：1万/渠道/年

### 5.3 实施费用
- **初次实施**：8-10万（根据客户环境复杂度调整）
- **定制开发**：1200元/人天

### 5.4 后续服务
- **维保服务**：3年后系统归属客户，维保费用0.75万/年
- **版本升级**：按需提供版本升级服务，费用另计

## 6. 交付标准

### 6.1 交付物
- **部署文档**：详细的系统部署和配置文档
- **操作手册**：用户操作指南和管理员手册
- **测试报告**：系统功能测试和性能测试报告
- **培训资料**：系统使用和维护培训课件

### 6.2 交付周期
- **标准交付**：从合同签订到系统上线15个工作日
- **定制交付**：根据定制内容评估交付周期，最长不超过30个工作日

## 7. 维护与支持

### 7.1 维护内容
- **系统监控**：定期系统性能监控和健康检查
- **故障修复**：及时修复系统故障和安全漏洞
- **性能优化**：根据系统运行情况进行性能优化
- **技术咨询**：提供技术咨询和最佳实践建议

### 7.2 支持流程
1. **问题提交**：客户通过电话、邮件或系统提交问题
2. **问题分级**：根据问题严重程度进行分级
3. **问题解决**：技术团队根据优先级解决问题
4. **问题反馈**：向客户反馈问题解决情况
5. **问题跟踪**：记录问题处理过程，形成知识库

## 8. 合同条款

### 8.1 合同期限
- 基础合同期限为3年，自系统上线之日起计算
- 合同期满后，客户可选择续约或转为维保服务

### 8.2 系统归属
- 合同期满3年后，系统所有权归属客户
- 客户享有系统的永久使用权和修改权

### 8.3 保密条款
- 双方需严格保密项目相关的技术文档和商业信息
- 保密期限为合同终止后5年

### 8.4 违约责任
- 任何一方违反合同条款，需承担相应的违约责任
- 具体违约责任根据实际损失情况协商确定
